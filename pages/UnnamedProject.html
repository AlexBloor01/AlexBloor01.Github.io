Task Management System

This task management structure chosen uses the Command Design Pattern and is ideal for games that feature a wide variety of tasks as well as multiple inputs, allowing each task to operate independently without being forced into a rigid framework. It offers flexibility in starting, completing, and canceling tasks while ensuring they are properly managed without unnecessary complexity.

The task management system operates as follows:

  • Task Class: An abstract base class defines the task structure, enforcing the implementation of essential methods and ensuring consistent task handling.
  • Task Manager: This script manages a queue of tasks, processing them sequentially and automatically starting the next task once the current one is completed. It also supports canceling the current task and clears the queue if needed.
  • Task Initialization: For instance, to create a WalkToPositionTask from CharacterLogic, initialize it with WalkToPositionTask walkToPositionTask = new WalkToPositionTask(this, destinationPosition, pathTraverser);, and add it to the TaskManager using taskManager.AddTask(walkToPositionTask);.

    • A* Pathfinding Algorithm

    The A* algorithm is a powerful tool for finding the shortest path between two points in a grid.

  • Initialize: Add the starting node to the open list. Set its mCost (distance from the start) to 0 and calculate its eCost using fCost = mCost + eCost, where eCost is the estimate to the goal.
  • Processing: While there are nodes in the open list:
    • Select Node: Choose the node with the lowest fCost as the current node.
    • Move to Closed List: Transfer the current node to the closed list, indicating it has been evaluated.
    • Evaluate Neighbors: For each neighbor of the current node:
      • If the neighbor is not walkable or is already in the closed list, skip it.
      • If the neighbor is not in the open list, add it, and calculate its mCost, eCost, and fCost costs.
      • If the neighbor is already in the open list, check if the current path to it is shorter by comparing mCost costs. Update its costs if necessary.
    • Goal Check: If the goal node is reached, trace the path back from the goal to the start to reconstruct the shortest path.
